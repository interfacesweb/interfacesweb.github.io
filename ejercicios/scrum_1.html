1 Introduction

We are going to learn some basic guiding principles to improve usability and accesibility of web sites. We will also analyze some web sites to check if they implement these guidelines. Finally, we will design a prototype (mockup) of a web site and we will generate a HTML template for that prototype.

We will use the Eduscrum framework. Students will work in teams and they will prepare a presentation to show what they have accomplished.

Each team will have a repository in GitHub to store the HTML code. The main branch will be gh-pages. This branch allows us to browse an HTML site in http://github.io.

2 Learning goals

  • Learn some key features about web usability and accesibility.
  • Structure web pages in a correct way.
  • Create web mockups / wireframes.
  • Organize a web page using DIV tags.
  • Use a template language (JADE) to create HTML layouts.
  • Use control version software to work in teams.

4 Product Backlog

4.1 Reading: Don't Make Me Think

Each team must read and prepare a summary of some chapters of the book Don't Make Me Think, Revisited, by Steven Krug. Each team will pick one of these options (each team MUST choose a different option):

  • Chapter 1, 2 & 3
  • Chapter 4, 5 & 6
  • Chapters 7, 8 & 9
  • Chapters 10, 11 & 12

Teams can divide the pages to read among the team members. Team members can then explain what they have read to the rest of the team in order to practice their oral English skills.

Teams must pick up 20 key words and look for its definition. They will create a glossary entry for each of them in the DIW Moodle course. Key words must be different from the other teams' key words.

Teams will read their summary in the demo class. Every member of the team must read aloud a piece of the summary. Remember you will have 20 minutes for the whole demo, not only the reading.

Definition of Done
You have read and understood the text. Team members have shared the information they have read. The team has prepared the summary. The team has created 20 key words in the glossary.

4.2 Website analysis

Each team must analyze one the proposed websites in terms of usability and navigation. Each team must pick a different site.

  • http://www.lingscars.com/
  • http://www.avis.es/
  • http://www.pcbox.com/
  • http://www.pccomponentes.com/

Teams will browse through the site and they will try to answer the following questions:

  • What site is this? (Is it clear enough?)
  • Does it follow conventions?
    • Menus, logos, content,…
    • Are elements clear enough? (Search inputs, icons, social networks,…)
  • Is the hierarchy evident (sections, subsections,…)? Can I see the major sections of the site? What options do I have at this level of the hierarchy?
  • Are the pages divided in well defined sections / layers?
  • Can you distinguish links and buttons?
  • Is there any content that distracts the user?
  • Is the content structured?
  • Are there any search options? Are they clear enough?

Each member of the team MUST analyze the website on his/her own and try to answer the questions. Later, the whole team can share what they have and prepare the final analysis.

Definition of Done
The team has chosen a website. Each team member has analyzed the website and answered the questions. The team has discussed and prepared the final analysis.

4.3 Prototyping

Teams will choose a free theme for a website (restaurant, school, store,…). They will prepare a mockup of the website using Pencil, available here. Alternate options are balsamiq and wireframe.cc.

Teams must try to follow the guidelines of Don't Make Me Think.

Each site must have 2 navigation levels (categories).

Definition of Done
The team has designed all the pages of the website. Each page only shows the appropriate elements of the list.

4.3.1 Pages to design

  • Home page
  • First level page
  • Second level page
  • Sign-up / sign-in form page
  • Location & contact information page
  • Contact form page

4.3.2 Elements to display (depending on the page)

  • Logo
  • Site name (company name, service name,…)
  • Page title
  • General search form
  • Main navigation menu
  • Secondary navigation menu
  • Location map
  • Telephone and address info
  • Home link
  • Important info (sales, notifications,…)
  • Ads
  • Login link

4.4 HTML Layout with PUG

Teams will create an HTML template using PUG for the site designed previously.

Definition of Done
Each team member has installed and read the documentation about PUG. The team has created the HTML templates using PUG. The whole website is deployed in the team repository and it is available in GitHub Pages.

4.4.1 Install PUG

  • Download node: https://nodejs.org/en/download/
  • Install PUG https://pugjs.org/api/getting-started.html#installation
  • Read some of the documentation
    • https://pugjs.org/language/doctype.html
    • https://pugjs.org/language/tags.html
    • https://pugjs.org/language/attributes.html

4.4.2 Template creation

  • Create a layout for each page you have defined in the prototype. Do not include the CSS yet. It will be done during the next sprint. Only HTML markup. Prepare some example content (Lorem ipsum dolor…): http://es.lipsum.com/
  • Deploy it in your Github team repository.
  • Preview it in github.io.
