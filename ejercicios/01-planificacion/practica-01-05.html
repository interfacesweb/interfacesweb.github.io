1 Introducción

El objetivo de la práctica es conocer la sintaxis básica de Markdown, un lenguaje de marcado ligero.

2 Tiempo asignado

1 hora de clase.

3 Material de la práctica

Los materiales para realizar esta práctica son los mismos que los de la práctica 1.4. Están disponibles en el siguiente enlace: Archivos práctica 1.5

4 Desarrollo de la práctica

Se debe modificar el código de cada una de las páginas para transformarlas en páginas HTML con las siguientes características:

  • El contenido debe organizarse en una estructura acorde con el texto. Para ello se utilizarán encabezados (pueden ser de distintos niveles) y párrafos.
  • Cada página debe disponer de un menú que permita navegar entre cada una de las 4 páginas que componen el sitio. El menú se diseñará como una lista desordenada.
  • Cada página debe tener remarcardos tres trozos de texto no consecutivos. El formato de énfasis puede ser negrita o cursiva. Dichos trozos de texto deben estar contenidos en un párrafo (es decir, no se puede seleccionar texto de dos o más párrafos distintos).

5 Resultado a entregar

  • Se deben entregar los cuatro ficheros HTML comprimidos en un sólo archivo.
  • El archivo estará en formato ZIP.
  • El nombre del archivo será de la forma apellido1-apellido2-nombre-practica1-5.zip (por ejemplo, prieto-alarcon-pedro-practica1-5.zip).
